Use for frontend changes only.
Before UI implementation, define or confirm the shared design elements for the task: layout grid, spacing, typography, color tokens, components, and states. Build from those primitives first to avoid hallucinated UI variants.
Load the smallest needed subset: delegate_task, structured_validation, document_sync, verification_before_completion.
Codex model policy: use configured native profiles.
