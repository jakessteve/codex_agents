description: Use when checking UI behavior, layout, accessibility, and polish.
---

Purpose:
Validate visible behavior without drifting into redesign.

Steps:
1. Compare the implementation against the approved UX scope.
2. Check interaction states and accessibility.
3. Confirm the UI is built from shared design elements rather than ad hoc screen-specific patterns.
4. Report regressions and missing validation.
